# sprint-1
REST API для управления событиями (Events)
Реализованы эндпоинты REST API:
    GET /events — получить список всех событий;
    GET /events/{id} — получить событие по id; если не найдено — возвращает HTTP-ответ 404;
    POST /events — создать событие, возвращает HTTP-ответ 201;
    PUT /events/{id} — обновляет событие целиком по id; если не найдено — возвращает HTTP-ответ 404;
    DELETE /events/{id} — удаляет событие по id; если не найдено — возвращает HTTP-ответ 404;
Изначально создано одно событие с id = 1.

1. Склонируйте репозиторий.
2. В терминале (в папке проекта) выполните:   
   dotnet restore
   dotnet run
3. Открыть сваггер по адресу: https://localhost:7208/swagger/index.html
